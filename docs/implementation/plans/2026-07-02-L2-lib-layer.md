## Tasks

### T1 lib/http:统一响应与错误码
- `error-codes.ts`:13 个错误码常量 + HTTP 状态映射(UNAUTHENTICATED:401/FORBIDDEN:403/APP_ACCESS_DENIED:403/USER_NOT_FOUND:404/APPLICATION_NOT_FOUND:404/VALIDATION_ERROR:400/CONFLICT:409/KEYCLOAK_ERROR:502/DEPENDENCY_FAILED:424/SYNC_PENDING:202/RATE_LIMITED:429/INTERNAL_ERROR:500/CSRF_REJECTED:403)。
- `api-error.ts`:`ApiError extends Error { code, status, details? }` + 工厂。
- `request-id.ts`:`newRequestId()`(`req_` + uuid)、`newTraceId()`、`newOperationId()`、`newEventId()`。
- `response.ts`:`ok(data, requestId)`、`fail(error, requestId)` → 设计约定 JSON 结构(camelCase)。
- 单测:错误码映射、响应结构、ApiError 序列化。

### T2 lib/crypto:PII 字段加密
- `field-encryption.ts`:AES-256-GCM;`encryptField(plain, key)` → `v1.<iv_b64>.<tag_b64>.<cipher_b64>`,`decryptField`;密钥来自 env `PII_ENCRYPTION_KEY`(32 字节 base64)。
- 单测:回环、密钥错误抛错、格式版本前缀、篡改检测(tag 校验失败)。

### T3 lib/permissions:纯函数权限评估器
- `types.ts`:`AdminGrant { roleCode, permissionCodes[], scopeType: 'global'|'org'|'app', scopeId }`、`PermissionScope { type: 'org'|'app', id }`。
- `evaluate.ts`:`can(grants, permissionCode, scope?)` 与 `canWithReason(...)`;规则:platform_admin 全权限;global 授权覆盖一切 scope;org 授权覆盖该 org 与其 app 范围校验(org→app 继承交由调用方传入映射,评估器只比对 scope 精确匹配 + global/org 覆盖);拒绝返回统一 FORBIDDEN 语义(不暴露缺哪个权限)。
- 单测:三层继承矩阵、越权拒绝、platform_admin 短路。DB 接线在 L3 policies。

### T4 lib/rate-limit:Redis 滑动窗口
- `sliding-window.ts`:`checkRateLimit(redis, key, { windowMs, max })` → `{ allowed, remaining? }`(实现不向调用方暴露 remaining 给终端用户);Key 规范 `rate_limit:<scene>:<dims>`(username 用 hash)。
- 集成测试(真实 Redis):窗口内第 max+1 次拒绝、窗口滑动后恢复。
- 依赖 ioredis;`lib/rate-limit/redis.ts` 惰性单例(REDIS_URL)。

### T5 lib/mq:adapter 契约 + RabbitMQ 实现
- `types.ts`:`MqAdapter { publish(topic, message, opts?): Promise<void>; consume(queue, handler, opts?): Promise<() => Promise<void>>; ack/nack 由 handler 返回值语义承担(resolve=ack, throw=nack+requeue 策略); healthCheck(): Promise<boolean>; close(): Promise<void> }`(决议 22:只抽象可靠传输)。
- `rabbitmq-adapter.ts`:amqplib;topic exchange `identity.events`(durable)+ 队列绑定 routing key;publisher confirm;消费 prefetch=10。
- 契约测试套件 `mq-adapter-contract.ts`(任何实现必须过)+ RabbitMQ 集成测试:发布→消费→ack;handler 抛错→nack 重投一次后丢弃(测试用队列 TTL/DLX 简化断言);healthCheck。

### T6 lib/audit + lib/keycloak
- `lib/audit/context.ts`:AsyncLocalStorage 承载 `{ requestId, traceId, operationId, actor }`;`runWithAuditContext(ctx, fn)`、`getAuditContext()`。(写入走 L1 audit-log-repository,由 L3 service 组装。)
- `lib/keycloak/token-client.ts`:client_credentials 获取 service account token + 过期前刷新缓存。
- `lib/keycloak/admin-client.ts`:基于 @keycloak/keycloak-admin-client 封装 `getKeycloakAdmin()`(token 自动续期);操作集:ensure/find 用户、disable/enable、logout 会话、重置密码(临时)、重置 MFA(删除 otp credential + requiredAction)、client role 授予/移除、按 clientId 查角色。
- `lib/keycloak/errors.ts`:HTTP 错误 → `KEYCLOAK_ERROR/USER_NOT_FOUND/CONFLICT` 映射。
- 集成测试(真实 KC):token 获取、建用户→disable→会话登出→client role 授予/移除→删除清理。

### T7 lib/auth:Auth.js v5 + Keycloak OIDC
- 依赖 next-auth@5(beta 线)。`lib/auth/auth-config.ts`:Keycloak provider(issuer=KEYCLOAK_BASE_URL/realms/REALM,client user-portal + secret env,`authorization: { params: { scope: 'openid email profile' } }`);callbacks:jwt(persist sub/realm_access.roles/access_token 有效期)、session(暴露 keycloakSub、roles、isAdmin=roles 含 admin_console_access);`trustHost: true`。
- `lib/auth/index.ts`:`export const { handlers, auth, signIn, signOut } = NextAuth(config)`。
- `app/api/auth/[...nextauth]/route.ts`:`export const { GET, POST } = handlers`。
- `lib/auth/require-admin.ts`:`requireAdmin()` → session 校验 + admin_console_access 角色,不满足抛 ApiError(UNAUTHENTICATED/FORBIDDEN)。
- 登录/登出 UI 最小件:`/login` 页(签入按钮→signIn('keycloak'))、门户首页显示会话状态与登出。
- env 扩展:AUTH_SECRET、KEYCLOAK_CLIENT_ID/SECRET(bootstrap 脚本输出 user-portal secret 的获取方式:confidential client secret 从 KC 读取并写 .env 提示——增加 `scripts/print-client-secret.ts`)。
- **登录冒烟(层验收)**:browser preview——/login → Keycloak 登录页(zh-CN)→ 种子管理员登录(临时密码首登改密)→ 回门户显示邮箱与登出按钮 → 登出回登录态。
