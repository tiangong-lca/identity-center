# L4 接口层 Implementation Plan

**Goal:** 全部管理/公共/账号端点 + 统一接线(CSRF/三层权限/审计上下文/幂等/限流/错误映射)+ 契约测试 + OpenAPI。

**Architecture:** `app/api/_helpers.ts`(app 元素,可依赖 lib-server/policies/services)提供三种路由包装器:
- `adminRoute({ permission, scope? }, handler)`:requestId→审计上下文→CSRF(写操作 Origin+JSON-only)→requireAdmin→requirePermission→限流(adminApi 100/min/user)→zod 校验→统一响应/错误映射;`Idempotency-Key` 重放守卫(processed_events,重复→CONFLICT)。
- `publicRoute({ scene })`:IP 限流(register 10/h 等)+ CSRF + zod。
- `accountRoute(handler)`:仅要求登录会话。

**可测性:** `services/context.ts` 增加 `__setServiceContextForTests()`;契约测试 vi.mock `@/lib/auth`(注入会话)+ 测试库 ServiceContext,直接调用 Route Handler(Request→Response),覆盖 401/403/400/409 与审计断言。

**端点组(实现顺序):** health → admin/users(8)→ registration(4+1 public)→ applications(4)+assignments(3)+roles(6)→ organizations(9)→ admin-roles/permissions(7 含绑定)→ audit(1)→ account(profile/apps/sessions)。
**服务补齐:** user-service.update(displayName)+ admin-client.listUserSessions。
**撤权 API 语义:** revoke outcome=revoked→200;projection_failed→502 KEYCLOAK_ERROR(错误体附 syncPending 细节);重复撤销→409。
**OpenAPI:** `docs/references/openapi.yaml` 手工维护(信封+错误码 components + 全路径 operations)。

**验收:** 契约测试全绿(每组含未登录 401/越权 403/参数 400/冲突 409);每个管理写操作产生审计记录断言;撤权按语义返回;lint/typecheck 绿;进度表更新。
