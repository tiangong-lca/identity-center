# L5 页面层 Implementation Plan

**Goal:** 约 24 个页面对照 15 个原型实现;i18n 全量资源化;明暗双主题;Keycloak 登录主题;Playwright e2e。

**分批执行(每批 commit):**
1. **B1 组件底座**:shadcn/ui 初始化(主题变量对齐设计库 token)、DataTable(TanStack Query 分页/筛选)、PageHeader、ConfirmDialog(危险操作二次确认)、StatusTag、FormDialog;TanStack Query Provider;features/ 目录 API client(fetch 封装信封解包)。
2. **B2 管理后台壳**:AdminShell(48px TopNav + 200px Sidebar,对照 orchestration-summary 的 shell 契约)、导航配置(i18n)、/admin 概览页(KPI 卡片)、admin 布局服务端权限守卫(requireAdmin → 无权限 redirect /403)。
3. **B3 用户管理组**:/admin/users 列表(搜索/筛选/分页/批量)、/admin/users/new、/admin/users/[id] 详情(Tabs:概览/安全/组织/应用/角色/会话/审计 + 危险操作区:禁用/重置密码/重置MFA)。
4. **B4 注册审批组**:/admin/registration-requests(列表+审批/拒绝弹窗)、/admin/registration-config(注册配置展示,配置项一期为环境级只读说明)。
5. **B5 应用管理组**:/admin/apps 列表、/admin/apps/[id] 详情(Tabs:基础信息/Keycloak Client/准入用户/角色目录/角色分配/同步状态)。
6. **B6 组织/角色权限/审计/设置**:/admin/orgs、/admin/roles(管理角色+权限矩阵)、/admin/audit(时间线列表+筛选)、/admin/settings(系统信息只读)。
7. **B7 门户与账号中心**:/(用户门户首页:我的应用宫格)、/account(资料)、/account/security(改密/MFA 引导跳 Keycloak 账户控制台)、/account/sessions、/403、error 页。
8. **B8 Keycloak 登录主题**:custom theme(login.html 原型对齐:品牌区+表单布局,zh-CN/en 消息),挂载到 compose(volume)+ bootstrap 设置 realm loginTheme。
9. **B9 e2e(Playwright)**:登录、用户禁用、准入授予/撤销、注册审批、语言切换、主题切换;双主题走查截图。

**约束:** 全部文案经 messages(zh-CN/en 同步);颜色仅用主题 token 类;服务端组件优先,客户端组件只在交互处;前端权限隐藏仅体验(服务端为准)。
